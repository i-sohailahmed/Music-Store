# Music Store SQL Analysis Project

## Project Overview

This project is an end-to-end SQL analysis of a digital music store database using PostgreSQL.

The objective of the project is to analyze customer purchasing behavior, sales performance, artist popularity, music genres, and revenue trends by solving real-world business problems using SQL.

The project demonstrates practical SQL skills including joins, subqueries, Common Table Expressions (CTEs), window functions, aggregate functions, and data analysis techniques.

---

#  Business Problem

A digital music store wants to better understand its customers, sales performance, and music catalog.

Using SQL, this project answers important business questions such as:

- Who is the company's best customer?
- Which country generates the highest number of invoices?
- Which city generates the highest revenue?
- Which artists are most popular?
- Which genres are most purchased in each country?
- Which customers spend the most money?
- Which songs are longer than the average track length?

The insights generated from these queries can help management improve marketing strategies, customer engagement, and business decisions.

#  Business Questions Solved

This project answers **11 real-world business questions**, including:

1. Identify the senior-most employee based on job title.
2. Find the country with the highest number of invoices.
3. Retrieve the top three highest invoice totals.
4. Identify the city generating the highest revenue.
5. Find the highest-spending customer.
6. List all Rock music listeners.
7. Identify the top 10 Rock artists based on the number of tracks.
8. Find tracks that are longer than the average track length.
9. Calculate the total amount spent by each customer on the best-selling artist.
10. Determine the most popular music genre in each country.
11. Identify the highest-spending customer in each country.
